# AutoCinco Visualizer碰撞预检查

## 当前检查假设

- 机器人外框：含Intake方向16.65英寸，宽13.03英寸。
- 额外安全边距：四周各1英寸。
- 实际检查外框：18.65 × 15.03英寸。
- 144坐标系场地边界：0–144英寸。
- 障碍物：Visualizer当前DECODE默认红蓝Goal多边形。

## 当前结果

旧路径不是一条可以直接声明“不会碰撞”的路径。按以上保守外框检查：

- 红方靠近`PUSH GATE 1/2`的位置会接近或进入红方Goal外框/场地边界。
- 蓝方起点、第一排靠墙取球、`PUSH GATE 1/2`和第三排靠墙取球会接近或进入蓝方Goal外框/场地边界。
- 一些“下一段t=0碰撞”是上一段终点已经贴边造成的重复报告。

这些风险来自旧路径形状，不是把射击方向旋转180°或140→144坐标校正造成的。迁移时没有
擅自把取球或推gate路径向场内移动。

## 接下来如何确认

1. 如果最终实测外框不同，先更新生成器中的`rWidth`和`rHeight`。
2. 如果Intake或其他机构会伸出底盘，把伸出量加入对应方向，或增加Safety Margin。
3. 逐段查看03、04、07、08、11以及蓝方起点。
4. 如果确认会撞，只修改对应联盟的`RedAutoCinco.java`或`BlueAutoCinco.java`编号点，
   再运行生成器；不要直接手改公共Route数据类。
5. 修改后运行：

   ```text
   tools/validate-auto-cinco-visualizer.mjs
   ```

几何检查无法覆盖软质场地变形、轮胎打滑、Pinpoint误差和机构摆动，因此最终必须低功率实测。
